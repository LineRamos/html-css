Tags em HTML

Em HTML5, podemos inserir os elementos da lista a seguir. Passe o mouse sobre os termos para descobrir qual tag deverá utilizar em cada caso (para isso deve usar a tag <abbr>).

Nessa página, algumas funções também podem aparecer caso coloque o cursor em cima do que procura (normalmente são termos que tem sublinhado pontilhado).

  • Parágrafos

  • Títulos

  • Imagens

  • Abreviações - também pode ser utilizado para colocar explicações como aparecem nesse desafio

  • EMOJIs - Para acrescentar, precisa iniciar com "&#x".
    Após esse começo, deve acrescentar o código referente ao emoji desejado.
    Para pesquisar, pode acessar o site EMOJIPEDIA.ORG
    Escolhe o emoji, clica nele e procura a aba "Technical Information" e acrescenta ao código, a informação "CODEPOINTS" (sem o U+): &#x + CODEPOINTS (escreve tudo junto).

  • Códigos-fonte - é uma forma de demonstrar o (s) código (s) que deseja compartilhar, porém com uma forma estética, onde é possível visualizar de pronto do que se trata. Como aqui por exemplo, onde a aparência da fonte é diferente.

  • Listas numeradas - também chamadas de LISTAS ORDENADAS.
    Por padrão, a lista é numérica, iniciada em 1. Porém, tem opções de escolher, adicionando o parâmetro "type=" que mostrará as opções.
    Cada item da lista deve estar denro da tag <li>.
    1. Um
    2. Dois
    3. Três

  • Listas com demarcadores - são listas onde não tem hierarquia. Por padrão já vem com marcador em CÍRCULOS ("disc"), mas pode mudar para QUADRADOS ("square") ou CÍRC. VAZIO ("circle").
    Cada item da lista deve estar denro da tag <li>.

  • Negrito - esse recurso dá mais significado (do ponto de vista de conteúdo para páginas HTML5). Normalmente, usam a tag <b>, porém essa aplicação não dá conteúdo ao texto da página HTML5.

  • Itálico - essa tag tem o nome de ênfase, justamente por ser um recurso que vai servir como um conteúdo a mais para o HTML5, ao invés da tag <i>.

  • Sublinhado - essa tag é considerada como não-semântica (pois não agrega conteúdo de fato ao HTML5).

  • Texto com letras maiores ou letras menores.

  • Marca-texto - a cor do marcado pode ser alterada no HTML (como quero alterar de forma pontual, como está em "texto"), mas também pode ser melhor arrumada com as CSS
    • Para isso, depois de "mark", acrescenta STYLE e escolhe "background-color" e a cor desejada.
    • Inclusive, para fazer esse recuo, utiliza a tag <dd>

  • Aparência de texto deletado, texto inserido ou texto sublinhado

  • Aparência de texto sobrescrito (exemplo: a2+b5) e texto subescrito (exemplo: H2O2).

  • Citação simples - além de ser uma forma de agregar um conteúdo para aquela página, serve como uma opção d uso das aspas.
    • Exemplo: Eu tudo posso naquele que me fortalece. (Filipenses 4:13)

  • Citações completas - o conceito da aplicação dessa tag é semelhante ao caso de citações de textos com mais de quatro linhas (de acordo com a ABNT).
    Caso deseje, pode colocar o parâmetro "cite=" onde pode informar o link de onde foi retirado ou para indicar onde pode comprar.

    • Exemplo:

    1 Ainda que eu falasse as línguas dos homens e dos anjos, e não tivesse amor, seria como o metal que soa ou como o sino que tine.
    2 E ainda que tivesse o dom de profecia, e conhecesse todos os mistérios e toda a ciência, e ainda que tivesse toda a fé, de maneira tal que transportasse os montes, e não tivesse amor, nada seria(1 Coríntios 13:1,2).


  • Lista de definição ("Definition list") - para fazer um dicionário (por exemplo), onde vai colocar um TERMO (tag <dt>) e a sua DEFINIÇÃO/DESCRIÇÃO (tag <dd>).

Essas duas listas e demais tags, podem ser misturadas de acordo com a necessidade de organização.

Outra informação interessante é saber como acrescentar os links que podem ser:

  • Links externos
  • Links internos
  • Links para download

Para serem criados, precisam de "âncoras" (que é a tag <a>). Onde nessas âncoras tem toda a referência para informar se o link é externo, interno ou para download.

OBS.: Caso n saiba ainda qual será o link, pode digitar "#", pois significa que esse link está vazio.

  • Links externos - <a href="https://gustavoguanabara.github.io" target="_blank" rel="external">
    Em href=" " coloca o site ***** target=" " coloca _blank para que a página do link abra em outra aba (fazendo com que o usuário não saia da minha página) ***** rel=" " coloca external para sinalizar que aquele link é externo.

  • Links internos - <a href="pag002.html "rel="next">
    Em href=" " coloca o local onde se encontra o arquivo da página (pode pressionar CTRL + ESPAÇO e consegue visualizar as opções) (recomenda-se que as páginas estejam em pastas organizadas de acordo com a hierarquia, o que pode ser visto com esse atalho citado antes) ***** rel=" " coloca next para sinalizar que aquele link interno está localizado depois daquela página. Caso fosse uma página anterior, estaria escrito "prev".

  • Links para download - tem duas formas de fazer o uso:
    • Para baixar um arquivo "normal" - exemplo: <a href="livro/Nome do livro.pdf" download="Nome do livro.pdf" type="application/pdf">

      • Em href=" " coloca o local onde se encontra o arquivo da página (pode pressionar CTRL + ESPAÇO e consegue visualizar as opções) (recomenda-se que as páginas estejam em pastas organizadas de acordo com a hierarquia, o que pode ser visto com esse atalho citado antes) ***** Em download=" " coloca o nome do arquivo com a extensão. ***** Em type=" " deve informar o tipo de mídia ou "media types" (para saber mais, olhar a OBS que vem logo abaixo).

    • Para baixar um arquivo compactado - exemplo: <a href="livro/Nome do arquivo compactado.rar" download="Nome do arquivo compactado.rar" type="application/zip">

      • Em href=" " coloca o local onde se encontra o arquivo compactado da página (pode pressionar CTRL + ESPAÇO e consegue visualizar as opções) (recomenda-se que os arquivos compactados estejam em pastas organizadas de acordo com a hierarquia, o que pode ser visto com esse atalho citado antes) ***** Em download=" " coloca o nome do arquivo com a extensão. ***** Em type=" " deve informar o tipo de mídia ou "media types" (para saber mais, olhar a OBS que vem logo abaixo).

      OBS.: Os media types são parâmetros utilizados para sinalizar que tipo de arquivo está disponibilizado para download (como aqui é o caso de downloads). Existe uma relação de "mediatypes" que é acessada no site do IANA (pode buscar também no Google por "iana assignments media types").
